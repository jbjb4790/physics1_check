선생님용

GitHub로 무료 HTML 링크 배포하기

이 폴더의 파일을 GitHub 저장소에 올리고 GitHub Pages를 켜면, 학생에게 보낼 수 있는 무료 웹사이트 주소가 생깁니다.

가장 쉬운 업로드 순서

  1. GitHub 계정 만들기: github.com에서 무료 계정을 만듭니다.
  2. 새 저장소 만들기: 예를 들어 physics-check라는 public repository를 만듭니다.
  3. 파일 업로드: 압축을 푼 뒤 index.html, lesson.html, teacher.html, data.js, app.js, styles.css를 저장소의 최상위(root)에 업로드합니다.
  4. Pages 켜기: 저장소 Settings → Pages에서 Source를 Deploy from a branch, Branch를 main, Folder를 /(root)로 설정하고 Save합니다.
  5. 링크 공유: 몇 분 후 https://깃허브아이디.github.io/physics-check/ 주소가 열립니다. 회차별 링크는 아래 형식을 사용합니다.
1회차: https://깃허브아이디.github.io/physics-check/lesson.html?session=1
2회차: https://깃허브아이디.github.io/physics-check/lesson.html?session=2
...
17회차: https://깃허브아이디.github.io/physics-check/lesson.html?session=17

수업 운영 팁

수업 직후 5분

공식 카드 1분, O/X 오개념 2분, 객관식 2분으로 운영합니다.

오답 태그 확인

결과 화면의 “복습 우선순위” 태그를 보고 다음 수업 도입 질문을 정합니다.

제출 방식

학생이 결과 리포트를 복사하여 카카오톡, 구글 클래스룸, LMS에 제출하게 합니다.

문항 수정

data.js 파일에서 각 회차의 questions, misconceptions를 수정하면 즉시 반영됩니다.

현재 사이트의 회차별 링크

GitHub Pages에 올린 뒤 이 페이지에서 열면, 아래 버튼의 링크도 자동으로 GitHub 주소 기준으로 복사됩니다.

무료 정적 사이트라 중앙 저장 기능은 없습니다. 자동 수집이 필요하면 Google Form/Sheet, Apps Script, Formspree 같은 별도 수집 도구를 붙이면 됩니다.
